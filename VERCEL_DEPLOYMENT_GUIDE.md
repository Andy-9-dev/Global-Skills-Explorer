# Vercel Deployment Guide

## Overview
This guide explains how to deploy the Global Skills Explorer to Vercel and troubleshoot common issues.

## Prerequisites
- GitHub account with the repository pushed
- Vercel account (free tier available)
- Node.js 16+ installed locally

## Deployment Steps

### 1. Connect Repository to Vercel
1. Go to [vercel.com](https://vercel.com)
2. Click "New Project"
3. Select "Import Git Repository"
4. Connect your GitHub account
5. Select the Global Skills Explorer repository
6. Click "Import"

### 2. Configure Build Settings
Vercel should auto-detect the following:
- **Framework**: Vite
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Install Command**: `npm install`

These are already configured in `vercel.json` and `vite.config.js`

### 3. Environment Variables
Add the following environment variables in Vercel project settings:


### 4. Deploy
Click "Deploy" and wait for the build to complete.

## Troubleshooting

### 404 Error on Page Refresh
**Problem**: Refreshing any page other than the home page returns 404

**Solution**: This is already fixed with `vercel.json` which rewrites all routes to `index.html` for client-side routing.

**Files involved**:
- `vercel.json` - Handles routing rewrites
- `vite.config.js` - Specifies build output directory

### Favicon Not Showing
**Problem**: Favicon doesn't appear in browser tab

**Solution**: The favicon is in `public/favicon.svg` and is automatically copied to `dist/` during build.

**Verify**:
1. Check that `public/favicon.svg` exists
2. Run `npm run build` locally
3. Verify `dist/favicon.svg` exists after build
4. Check browser cache (Ctrl+Shift+Delete)

### Environment Variables Not Working
**Problem**: App shows errors about missing Supabase keys

**Solution**: 
1. Go to Vercel Project Settings в†’ Environment Variables
2. Add all `VITE_*` variables
3. Redeploy the project (Vercel в†’ Deployments в†’ Redeploy)

### Build Fails
**Problem**: Build fails during deployment

**Solution**:
1. Check build logs in Vercel dashboard
2. Run `npm run build` locally to reproduce
3. Ensure all dependencies are in `package.json`
4. Check for TypeScript/ESLint errors

## Key Configuration Files

### vercel.json
Handles SPA routing by rewriting all requests to `index.html`:
```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

### vite.config.js
Specifies build output and public directory:
```javascript
export default defineConfig({
  plugins: [react()],
  build: {
    outDir: 'dist',
    emptyOutDir: true,
    sourcemap: false,
  },
  publicDir: 'public',
})
```

## Performance Tips

1. **Code Splitting**: Consider using dynamic imports for large components
2. **Image Optimization**: Compress images before adding to public/
3. **Bundle Analysis**: Use `npm run build` and check bundle size
4. **Caching**: Vercel automatically caches assets with content hashing

## Monitoring

After deployment:
1. Check Vercel Analytics dashboard
2. Monitor error logs in Vercel
3. Test all routes on the deployed site
4. Verify dark mode toggle works
5. Test map with 120 job locations
6. Check favicon appears in browser tab

## Rollback

If deployment has issues:
1. Go to Vercel Deployments
2. Find the previous working deployment
3. Click "Promote to Production"
